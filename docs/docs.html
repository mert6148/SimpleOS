Overview

SimpleOS is a minimalist but fully-featured x86 operating system kernel, designed as an educational platform to understand core OS concepts.

Multitasking

Priority-based scheduler with 6 priority levels and round-robin execution within each level.

Memory Management

Buddy allocator for physical memory and virtual paging with 4KB pages.

File System

Virtual File System (VFS) with inode-based abstraction and RAMFs support.

Device Drivers

IRQ-driven driver framework with PIC and PIT support for interrupts and timing.

x86 Architecture

Full x86 boot sequence, GDT/IDT setup, exception handling, and paging.

Educational

Well-documented codebase with comprehensive guides for learning and extension.

Getting Started

Requirements

  • GCC cross-compiler (x86 target)
  • GNU Make
  • QEMU (for emulation)
  • Standard C library headers

Quick Start

$ make              # Build simpleos.elf
$ make qemu         # Run in QEMU
$ make clean        # Clean artifacts
$ make all          # Full rebuild

Architecture

SimpleOS uses a modular architecture with clear separation of concerns.

Scheduler

Location: kernel/core/sched.c

Priority-based scheduler with 6 levels (0-5). Uses round-robin within each priority level. Manages task states: CREATED → READY → RUNNING → BLOCKED → ZOMBIE.

scheduler_init()
scheduler_add_task()
scheduler_context_switch()

Memory Manager

Location: kernel/mm/mm.c

Buddy allocator for physical memory (power-of-2 block sizes) and virtual memory paging with 4KB pages. Maximum heap: 128MB.

kmalloc()
kfree()
vm_map() / vm_unmap()

Virtual File System

Location: kernel/fs/vfs.c

Inode-based abstraction supporting multiple file systems. Implements standard file operations: open, read, write, close. RAMFs backend included.

vfs_open()
vfs_read() / vfs_write()
vfs_close()

Device Drivers

Location: kernel/drivers/

IRQ-driven framework with PIC (Programmable Interrupt Controller) and PIT (Programmable Interval Timer). Supports interrupt registration and handling.

driver_register()
driver_unregister()
handle_irq()

x86 Architecture

Location: kernel/arch/x86/

Complete x86 boot sequence: real mode → protected mode. Sets up GDT, IDT, exception handlers, and paging with 4-level page tables.

boot.S
setup.c (GDT/IDT)
exceptions.S

Boot Sequence

Location: kernel/core/kmain.c

6-phase initialization: (1) Memory Setup, (2) Scheduler, (3) VFS, (4) Drivers, (5) Virtual Memory, (6) Init Task.

kmain()
Phase-specific inits
printk() output

Key Features

Priority-Based Scheduling

6 priority levels (0-lowest to 5-highest) with round-robin execution within each level. Enables multitasking with predictable task behavior.

Task States:
CREATED → READY → RUNNING
  ↓
BLOCKED → ZOMBIE

Buddy Allocator

Efficient physical memory management using power-of-2 block sizes. Prevents fragmentation and provides fast allocation/deallocation.

void *ptr;
KERNEL_STATUS status = kmalloc(256, &ptr);
if (status == KERNEL_SUCCESS) {
    // Use memory
    kfree(ptr);
}

Virtual Paging

4-level x86-64 page tables with 4KB pages. Enables virtual memory, process isolation, and memory protection.

// Map virtual to physical address
vm_map(virt_addr, phys_addr, FLAGS);

// Unmap virtual address
vm_unmap(virt_addr);

VFS Abstraction

Inode-based file system interface supporting multiple backends. RAMFs included as default in-memory file system.

struct file *f;
vfs_open("/file.txt", &f);
vfs_read(f, buffer, 512, &bytes_read);
vfs_write(f, data, size, &bytes_written);
vfs_close(f);

IRQ Driver Framework

Abstraction for registering interrupt handlers. Supports PIC (Programmable Interrupt Controller) for managing up to 16 IRQs.

struct driver_ops ops = {
    .init = driver_init,
    .handle_irq = irq_handler,
};
driver_register("device", IRQ_NUM, &ops);

Serial Console

Full printf-style logging via printk() to serial port (0x3F8). Essential for debugging and understanding OS behavior.

printk("[SCHED] Task PID=%d state=%d\n",
       task->pid, task->state);
printk("[MM] Allocated %d bytes at 0x%x\n",
       size, (u32)ptr);

API Reference

Quick reference for commonly used kernel APIs.

Memory Management

KERNEL_STATUS kmalloc(size_t size, void **ptr)

Allocate memory from kernel heap using buddy allocator. Returns KERNEL_SUCCESS on success, KERNEL_ENOMEM on failure.

KERNEL_STATUS kfree(void *ptr)

Free previously allocated memory. Returns KERNEL_SUCCESS if pointer was valid.

KERNEL_STATUS vm_map(addr_t virt, addr_t phys, u32 flags)

Map virtual address to physical address with specified page flags (present, writable, user, etc.).

KERNEL_STATUS vm_unmap(addr_t virt)

Unmap virtual address and remove page table entry.

Scheduler

KERNEL_STATUS scheduler_init(void)

Initialize scheduler, task queues, and TCB (Task Control Block) pool. Called during boot phase 2.

KERNEL_STATUS scheduler_add_task(struct task *task)

Add task to appropriate ready queue based on priority level. Transitions task to READY state.

void scheduler_context_switch(void)

Save current task context, pick next task from ready queue, and restore its context.

KERNEL_STATUS scheduler_block_task(struct task *task)

Transition task from RUNNING to BLOCKED state and trigger context switch.

Virtual File System

KERNEL_STATUS vfs_open(const char *path, struct file **file)

Open file at given path. Creates file handle with file operations from registered file system.

KERNEL_STATUS vfs_read(struct file *file, void *buf, size_t size, size_t *read)

Read up to size bytes from file. Returns actual bytes read in *read parameter.

KERNEL_STATUS vfs_write(struct file *file, const void *buf, size_t size, size_t *written)

Write up to size bytes to file. Returns actual bytes written in *written parameter.

KERNEL_STATUS vfs_close(struct file *file)

Close file handle and flush any pending data.

Device Drivers

KERNEL_STATUS driver_register(const char *name, u32 irq, struct driver_ops *ops)

Register device driver with interrupt number and operations. Enables interrupt routing to handler.

KERNEL_STATUS driver_unregister(const char *name)

Unregister driver and disable interrupt routing.

void handle_irq(u32 irq_number)

Main IRQ handler. Routes interrupt to registered driver handler via PIC (Programmable Interrupt Controller).

Type System

SimpleOS uses shorthand type definitions (kernel_types.h) for portability:

s8, s16, s32, s64 Signed integers (8, 16, 32, 64-bit)
u8, u16, u32, u64 Unsigned integers (8, 16, 32, 64-bit)
bool Boolean type (True/False)
KERNEL_STATUS Status/error code (0=success, negative=error)
addr_t Virtual/physical address type

Status Codes:

KERNEL_SUCCESS (0) - Operation succeeded
KERNEL_ENOMEM (-4) - Memory allocation failed
KERNEL_EINVAL (-7) - Invalid argument
KERNEL_EBUSY (-16) - Resource busy

Code Examples

Basic Memory Allocation

#include "kernel/include/kernel_types.h"
#include "kernel/include/mm.h"

void *ptr;
KERNEL_STATUS status = kmalloc(1024, &ptr);

if (status != KERNEL_SUCCESS) {
    printk("Allocation failed\n");
    return status;
}

// Use memory at ptr
printk("Allocated 1024 bytes at 0x%x\n", (u32)ptr);

// Free when done
kfree(ptr);

Creating a Task

#include "kernel/include/sched.h"

struct task *task = kmalloc(sizeof(struct task), (void**)&task);
task->pid = next_pid++;
task->priority = 3;  // Mid-priority
task->state = TASK_CREATED;
task->entry_point = my_task_function;

// Add to scheduler
scheduler_add_task(task);

void my_task_function() {
    printk("Task running!\n");
}

File Operations

#include "kernel/include/fs.h"

struct file *f;
KERNEL_STATUS status = vfs_open("/file.txt", &f);

if (status == KERNEL_SUCCESS) {
    char buffer[256];
    size_t read;
    
    status = vfs_read(f, buffer, 256, &read);
    printk("Read %d bytes\n", read);
    
    vfs_close(f);
}

Device Driver Registration

#include "kernel/include/driver.h"

static KERNEL_STATUS my_driver_init(void) {
    printk("[DRIVER] Initializing...\n");
    return KERNEL_SUCCESS;
}

static void my_irq_handler(u32 irq) {
    printk("[DRIVER] IRQ %d handled\n", irq);
}

KERNEL_STATUS my_driver_register(void) {
    struct driver_ops ops = {
        .init = my_driver_init,
        .handle_irq = my_irq_handler,
    };
    
    return driver_register("my_device", 5, &ops);
}
